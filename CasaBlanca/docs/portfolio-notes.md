# Notas para Portafolio

## Como presentar el proyecto

Casa Blanca se puede presentar como una plataforma web de reservas hoteleras desarrollada para demostrar dominio de backend, frontend clasico y modelado relacional.

## Que vale la pena destacar

- Implementacion completa del patron MVC.
- Integracion real con SQL Server.
- Formularios con validaciones.
- Autenticacion con control de roles.
- Dashboard administrativo con metricas.
- Gestion de hoteles y reservas dentro del mismo dominio de negocio.

## Enfoque recomendado para tu portafolio

Cuando lo describas en GitHub, LinkedIn o tu CV, puedes resumirlo asi:

> Desarrolle una aplicacion web de reservas hoteleras con ASP.NET MVC 5, C# y SQL Server. Implemente autenticacion de usuarios, panel administrativo, catalogo de hoteles, registro de reservas y persistencia con ADO.NET mediante consultas parametrizadas.

## Que puedes mostrar en una demo

- Registro de usuario.
- Inicio de sesion como cliente.
- Listado de hoteles.
- Creacion de una reserva.
- Consulta de reservas del usuario.
- Inicio de sesion como administrador.
- Dashboard admin.
- Alta y edicion de hoteles.

## Que decir como mejoras futuras

- Endurecer seguridad de autenticacion.
- Completar gestion administrativa de usuarios y reservas.
- Incorporar pruebas automatizadas.
- Desplegar la aplicacion en un entorno productivo.
- Mejorar busqueda y filtros del catalogo.

## Lectura honesta del proyecto

El proyecto ya comunica bien una base solida de desarrollo full stack tradicional. Para que se vea aun mejor en portafolio, conviene mostrarlo como una solucion funcional con roadmap de mejoras, no como un producto completamente terminado.
